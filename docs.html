TRaQ Attendance System

Technology-Driven Recording and Automated QR Code System for Attendance Monitoring and Logging — built on ESP32, Firebase, and Telegram.

Version 1.0.0-beta Author: Darren Bardelas Philippines 🇵🇭 2026

Overview

TRaQ is a fully automated, self-contained QR code attendance monitoring system designed for Philippine schools. Students scan their personal QR ID card at an ESP32-powered kiosk. The system logs attendance in real-time to Firebase Realtime Database and immediately notifies parents and teachers via Telegram bot — with no server or monthly fees.

The system is engineered to be resilient: it continues operating without internet by saving scans to onboard flash memory, then automatically syncing when connectivity is restored.

Student scans QR card ↓ GM65 scanner reads JSON payload via UART ↓ ESP32 parses student data + determines phase ↓ WiFi available? ──No──→ SPIFFS queue (offline storage) ↓ Yes Firebase RTDB attendance record written ↓ Telegram Bot notifies parent instantly ↓ LCD shows name + status | LED flashes

Features

Core Attendance

📷

4-Phase Scanning

Morning In/Out and Afternoon In/Out per student per day, with duplicate scan protection.

⚠️

Late Detection

Automatic late marking for students who scan after the configurable cutoff time.

🔴

Absent Marking

Automatic absent marking at 8:30 AM (morning) and 2:00 PM (afternoon) via scheduled triggers.

📊

Daily Finalization

7:30 PM summary sent to every parent and teachers receive a full section report.

Reliability

📶

Offline Mode

Scans saved to SPIFFS flash memory. Auto-syncs to Firebase when WiFi returns.

🔄

Multi-WiFi Failover

Up to 4 WiFi networks with best-signal selection. Non-blocking reconnect keeps scan loop running.

🕐

RTC + NTP

DS3231 RTC for accurate offline time. NTP syncs on every boot. Battery health monitored every 5 minutes.

⚡

Catch-up Checks

On boot, ESP32 checks systemFlags and triggers any missed absent checks via Apps Script.

Firebase Setup

1
Create Project
Go to console.firebase.google.com → Add project → name it (e.g. traq-attendance) → disable Analytics → Create project.
2
Realtime Database
Build → Realtime Database → Create Database → choose region asia-southeast1 → start in Test mode → copy the Database URL.
3
Get API Key
Project Settings ⚙️ → General → Your apps → click </> → register web app → copy the apiKey value.
4
Enable Anonymous Auth
Build → Authentication → Get started → Sign-in method tab → Anonymous → Enable → Save.
Test Mode expires after 30 days. Lock your database rules with a Firebase Secret before going live in production. See the Security section for the exact rules.

QR Code Format

Each student QR code must encode a JSON string with these exact fields. The LRN (L) is used as the unique Firebase key and must be exactly 12 digits.

json
{
  "N":  "Juan Dela Cruz",        // Full student name
  "L":  "123456789012",          // LRN — exactly 12 digits
  "G":  "Grade 12",              // Grade level
  "S":  "STEM",                  // Section name
  "PP": "+63 912 345 6789",      // Parent phone
  "P":  "+63 912 345 6789"       // Student phone
}

Attendance Schedule

Before 8:30 AM Scan 1st scan → Morning In (On Time)
After 8:30 AM Scan 1st scan → Morning In (Late)
8:30 AM Auto Students without morning scan → Absent AM — parent and teacher notified
~11:00 AM Scan 2nd scan → Morning Out
After 12:30 PM Scan 3rd scan → Afternoon In (too early before 12:30 PM)
2:00 PM Auto Students without afternoon scan → Absent PM — parent and teacher notified
Anytime PM Scan 4th scan → Afternoon Out
7:30 PM Auto Daily finalization — parent summary + full teacher section report with present/absent/late counts

Scan Phase Logic

The system checks Firebase state to determine which of 4 phases applies to each scan:

Student scans QR │ ├── No morningIn AND not absentAM? │ └── SCAN 1: MORNING IN (ON TIME or LATE based on time) │ ├── Has morningIn AND no morningOut? │ └── SCAN 2: MORNING OUT │ ├── (Has morningOut OR absentAM) AND no afternoonIn AND not absentPM? │ ├── Before 12:30 PM → "Too early!" │ └── After 12:30 PM → SCAN 3: AFTERNOON IN │ ├── Has afternoonIn AND no afternoonOut? │ └── SCAN 4: AFTERNOON OUT │ ├── absentAM AND absentPM? │ └── "Absent Today" │ └── All complete? └── "All Done!"
Special case: If a student was marked Absent AM but arrives in the afternoon, the system detects absentAM=true, skips morning phases, and goes directly to Afternoon In. The absentPM flag is also cleared.

Offline Mode

When WiFi is unavailable, TRaQ continues accepting scans. Entries are saved to /traq_queue.json on ESP32 internal flash and automatically uploaded when connectivity returns.

FeatureDetail
Storage locationESP32 SPIFFS flash (/traq_queue.json)
Survives rebootsYes — data persists through power cuts
Capacity~50–100 scans (16KB buffer)
Phase detectionSaved at scan time using current clock — no wrong-phase on flush
Duplicate protectionSame LRN + same date is rejected with red LED flash
Key collisionKeys use millis() + random(100000) — essentially impossible to collide
Auto-sync triggerDetected on next loop iteration when WiFi reconnects
LCD during offlineShows [OFFLINE] instead of scrolling text
